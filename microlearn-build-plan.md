# MicroLearn — Build Plan (Cursor Implementation Spec)

A daily micro-learning app: users pick interests, and every day they get a beautifully designed, animated stack of 4–6 cards that teaches one concept in ~60 seconds, generated by Claude.

---

## 1. Tech Stack

| Layer | Choice | Notes |
|---|---|---|
| Framework | Next.js 15 (App Router, TypeScript) | Server components + route handlers |
| Styling | Tailwind CSS v4 | Plus CSS variables for per-category themes |
| Animation | Framer Motion | Card transitions, staggered reveals, spring physics |
| Database | PostgreSQL + Prisma | SQLite locally is fine; Postgres (Neon/Supabase) in prod |
| AI | Anthropic API (`claude-sonnet-4-6`) | Structured JSON output for lessons |
| Auth | NextAuth v5 (email magic link or Google) | Keep minimal; single-user mode acceptable for v0 |
| State | React Server Components + small Zustand store for the card viewer | No Redux |
| Deployment | Vercel | Cron via Vercel Cron for daily generation |

## 2. Data Model (Prisma)

```prisma
model User {
  id          String   @id @default(cuid())
  email       String   @unique
  name        String?
  depth       Depth    @default(INTERMEDIATE) // BEGINNER | INTERMEDIATE | ADVANCED
  streak      Int      @default(0)
  lastDropAt  DateTime?
  interests   Interest[]
  drops       Drop[]
  savedCards  SavedCard[]
  createdAt   DateTime @default(now())
}

model Interest {
  id        String  @id @default(cuid())
  userId    String
  user      User    @relation(fields: [userId], references: [id])
  category  String  // e.g. "System Design", "GenAI", "Space", "Psychology"
  active    Boolean @default(true)
  @@unique([userId, category])
}

model Drop {
  id          String   @id @default(cuid())
  userId      String
  user        User     @relation(fields: [userId], references: [id])
  date        DateTime @db.Date
  category    String
  topic       String   // used for anti-repetition history
  lessonJson  Json     // full structured lesson (cards, quiz)
  completed   Boolean  @default(false)
  quizCorrect Boolean?
  createdAt   DateTime @default(now())
  @@unique([userId, date])
}

model SavedCard {
  id        String   @id @default(cuid())
  userId    String
  user      User     @relation(fields: [userId], references: [id])
  dropId    String
  cardIndex Int
  cardJson  Json
  createdAt DateTime @default(now())
}

enum Depth { BEGINNER INTERMEDIATE ADVANCED }
```

## 3. Lesson JSON Schema (the contract everything depends on)

Claude must return EXACTLY this shape. Validate with Zod (`lib/schemas/lesson.ts`) and reject/retry on failure.

```ts
import { z } from "zod";

export const VisualMotif = z.enum([
  "orbit",        // orbiting particles — space, cycles, ecosystems
  "flowNodes",    // animated node graph — networks, systems, pipelines
  "pulseBars",    // rising/pulsing bars — finance, growth, metrics
  "wave",         // sine waves — signals, audio, physics, emotion
  "stackBlocks",  // stacking blocks — layers, architecture, composition
  "branch",       // branching tree — decisions, evolution, git
  "spark",        // radiating sparks — ideas, neurons, energy
  "gridReveal",   // grid cells lighting up — data, memory, matrices
  "spiral",       // expanding spiral — compounding, recursion, time
  "morphBlob",    // morphing organic blob — biology, abstraction, change
]);

export const Card = z.object({
  type: z.enum(["hook", "concept", "example", "takeaway"]),
  title: z.string().max(60),
  body: z.string().max(320),          // ~40–60 words
  emphasis: z.string().max(80).optional(), // one phrase to highlight in large type
  visual: VisualMotif,
});

export const Lesson = z.object({
  category: z.string(),
  topic: z.string().max(80),
  cards: z.array(Card).min(4).max(6), // order: hook → 1–3 concept → example → takeaway
  quiz: z.object({
    question: z.string().max(160),
    options: z.array(z.string().max(80)).length(3),
    correctIndex: z.number().int().min(0).max(2),
    explanation: z.string().max(200),
  }),
});
export type Lesson = z.infer<typeof Lesson>;
```

## 4. AI Generation

### 4.1 Endpoint
`POST /api/drops/generate` (route handler, server-only):
1. Pick today's category: rotate through user's active interests (round-robin by drop count per category).
2. Fetch the last 30 `topic` values for that user+category → anti-repetition list.
3. Call Anthropic API with the system prompt below; parse + Zod-validate; retry up to 2x on schema failure (feed the validation error back in the retry message).
4. Persist as `Drop` (unique per user per day — return existing drop if already generated).

### 4.2 System prompt (store in `lib/prompts/lesson.ts`)

```
You are a world-class micro-learning content designer. Generate ONE 60-second
lesson as JSON matching the provided schema. Rules:
- ONE idea total. Each card carries exactly one beat of that idea.
- Card body: 40–60 words, conversational, zero filler, no bullet points.
- Hook card: a surprising question or counterintuitive fact.
- Example card: a concrete, relatable analogy (everyday life, not jargon).
- Takeaway card: one memorable sentence the reader can repeat tomorrow.
- "emphasis": the single most quotable phrase from that card's body, verbatim.
- Pick the visual motif that best matches each card's content from the enum.
- Depth: {{depth}}. Category: {{category}}.
- Do NOT use any of these past topics: {{pastTopics}}.
- Respond with ONLY the JSON object. No markdown fences, no preamble.
```

API call: `max_tokens: 2000`, temperature default. Use the official `@anthropic-ai/sdk`. API key in `ANTHROPIC_API_KEY` env var, never exposed client-side.

### 4.3 Daily cron
`vercel.json` cron → `GET /api/cron/daily` (protected by `CRON_SECRET` header check) → pre-generates drops for all users so the morning open is instant.

## 5. UI / Pages

### 5.1 Routes
- `/onboarding` — interest picker (chip grid, pick 3–5) + depth selector. Writes to DB, redirects to `/`.
- `/` (Today) — today's drop as the animated card stack viewer. If not generated yet, show a "brewing" animation while calling generate endpoint.
- `/library` — saved cards grid (masonry), filter by category.
- `/streak` — simple streak + history calendar (can be a section on `/` for v0).

### 5.2 Card Stack Viewer (the centerpiece) — `components/CardStack.tsx`
- Full-viewport vertical card, swipe horizontally (or arrow keys / click zones) to advance. Framer Motion `AnimatePresence` with spring exit/enter (`x` translate + slight rotate, like a deck).
- Progress dots top-center; tap-and-hold pauses ambient animation.
- Text choreography: title fades up first, body staggers in by line (`staggerChildren: 0.06`), `emphasis` phrase rendered in display type with a gradient highlight sweep.
- Last card → quiz card → result + streak increment animation (confetti only on streak milestones: 3, 7, 14, 30).
- "Save" (bookmark icon) on every card → `SavedCard`.
- "Share as image" (v1, not v0): render card to PNG via `html-to-image`.

### 5.3 Animated visual motifs — `components/motifs/`
One React component per `VisualMotif` enum value. Rules:
- Pure SVG + Framer Motion (or CSS keyframes) — NO external images, NO canvas libs.
- Each motif sits as the card's background layer at ~25% opacity with a subtle parallax (moves a few px opposite to swipe direction).
- Each accepts a `palette` prop (from the category theme) and respects `prefers-reduced-motion` (fall back to static SVG).
- Keep each under ~120 lines; loop animations at 6–10s, eased, never frantic.

### 5.4 Theming — `lib/themes.ts`
Map category → theme: `{ gradient: [from, to], accent, motifPalette }`. Examples:
- GenAI: deep violet → electric blue, accent cyan
- Finance: emerald → teal, accent gold
- Space: midnight navy → purple, accent white/star
- Psychology: warm coral → magenta, accent cream
Fallback theme for unknown categories (hash category name → pick from 8 preset palettes). Dark, rich backgrounds; light text; large display font (e.g., `Instrument Serif` or `Fraunces` for emphasis, `Inter` for body).

## 7. Build Phases (Cursor task order)

**Phase 1 — Skeleton (no AI yet)**
1. Scaffold Next.js + Tailwind + Prisma + schema + migrate.
2. Build `CardStack` viewer with a hardcoded mock `Lesson` JSON.
3. Build 4 motifs first: `orbit`, `flowNodes`, `pulseBars`, `spark`. Stub the rest as `gridReveal` fallback.
✅ Acceptance: can swipe through a mock 5-card lesson with animations at 60fps, reduced-motion respected.

**Phase 2 — AI generation**
4. Zod schema + Anthropic client + generate endpoint with validation/retry + persistence.
5. Wire Today page: fetch-or-generate today's drop.
✅ Acceptance: fresh user gets a valid generated lesson < 8s; invalid JSON auto-retries; same-day refresh returns cached drop.

**Phase 3 — Loop mechanics**
6. Onboarding interest picker + depth.
7. Quiz card + completion endpoint + streak logic + library save/view.
✅ Acceptance: completing a drop increments streak once per day; saved cards appear in library with correct theme.

**Phase 4 — Polish**
8. Remaining 6 motifs, theming pass, empty/loading states ("brewing your drop" animation), mobile swipe gestures, PWA manifest.
9. Cron pre-generation + auth.
✅ Acceptance: Lighthouse mobile ≥ 90 performance; works installed as PWA.

## 8. Guardrails for Cursor

- Never call the Anthropic API from client components. Server route handlers only.
- All lesson rendering must tolerate 4, 5, or 6 cards.
- Enforce `@@unique([userId, date])` — generation must be idempotent per day.
- Word-count limits live in the Zod schema, not just the prompt — truncate gracefully if Claude overruns slightly (don't fail the whole drop on a 65-word body; only fail on structural errors).
- No `localStorage` for core state; DB is the source of truth (streak especially).
- Keep motif components dependency-free (SVG + Framer Motion only).
- Respect `prefers-reduced-motion` everywhere.
